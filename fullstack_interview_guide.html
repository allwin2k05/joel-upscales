1. System Architecture Overview

The application is architected as a **Serverless Single Page Application (SPA)**. Rather than hosting a dedicated backend server (e.g., Express on Node.js) that requires ongoing maintenance, routing logs, and API hosting, the project relies on **Backend-as-a-Service (BaaS)** provided by **Supabase**.

Architectural Flow

  1. Client Request: The browser client downloads optimized React assets from Vercel's global CDN.
  2. Auth Validation: On load, the custom AuthContext verifies active sessions using JWT tokens stored in localStorage.
  3. Routing & Guarding: React Router matches the endpoint. If the user is unauthenticated, the ProtectedRoute redirects them to /login.
  4. Data Fetching: Component hooks invoke the Supabase JS Client, making secure CORS-approved REST requests to PostgreSQL.
  5. Mailing Services: Contact forms invoke the client-side EmailJS API directly, forwarding email messages to SMTP servers.

3. Database Schema & Seeding (PostgreSQL)

Our database engine runs on **PostgreSQL**. Unlike NoSQL engines, PostgreSQL enforces relational models and guarantees **ACID compliance** (Atomicity, Consistency, Isolation, Durability).

4. Authentication Flow & Client-Side Route Guards

Authentication utilizes secure **JWT tokens**. The state is managed via React's Context API in `AuthContext`, and route protection is enforced client-side before render.

// ProtectedRoute Component logic: const ProtectedRoute: React.FC<{ children: React.ReactNode; adminOnly?: boolean }> = ({ children, adminOnly }) => {
  const { isLoggedIn, isAdmin, loading } = useAuth();
  const location = useLocation();

  if (loading) return <LoadingSpinner />;

  if (!isLoggedIn) {
    return <Navigate to="/login" state={{ from: location }} replace />;
  }

  if (adminOnly && !isAdmin) {
    return <Navigate to="/" replace />;
  }

  return <>{children}</>;
};

5. UI/UX Graphics & CSS Glassmorphism

To set our app apart visually, we implemented premium glassmorphism classes and an interactive physics-based canvas background.

Glass Panel CSS

Using `backdrop-filter: blur(16px)` to achieve blurred background panels. Combined with `inset 0 1px 1px 0 rgba(255,255,255,0.05)`, it injects a 1px bright top-border, simulating reflection and premium styling.

Canvas Dot Animation

A background canvas monitors mouse movements. Using vector geometry, each dot calculates its relative distance from the pointer and accelerates away (repulsion) when inside the radius. The rendering loops inside `requestAnimationFrame` to match the native refresh rate, optimizing GPU performance.

6. Core Technical Interview Questions & Answers

These are direct, target-matched questions and professional responses to prepare you for the technical evaluation:

Q1: What is a JSON Web Token (JWT) and how does our application use it?

A JWT is a stateless, encoded string containing claims (such as user ID, role, and email) verified by a cryptographic signature. When a user logs in, Supabase generates the token. The browser caches it in localStorage. For all database transactions, the token is included in the Authorization header. Supabase decrypts and verifies the signature using the server's private key to authorize the transaction instantly without checking stateful tables.

Q2: Can you explain the client-side router 404 issue and how your vercel.json rewrites rule fixes it?

In a React single-page app, routing is handled client-side. When navigating to /about, React Router updates the address bar without talking to the server. However, if the user hits refresh on /about, the browser requests the file /about from the Vercel hosting server, which doesn't exist, throwing a 404. The vercel.json rewrite rule fixes this by redirecting all requests to /index.html, allowing React Router to boot up, parse the path, and mount the correct component.

Q3: Why did we experience a black screen crash on login, and how does React handle unhandled runtime errors?

The login page entered a loading state, which rendered the <Loader2 /> component. However, the import statement for Loader2 was missing from lucide-react. This triggered a ReferenceError at runtime. In React 16+, unhandled runtime errors during the render phase cause React to unmount the entire component tree (yielding a blank black screen) to prevent corrupted user interfaces. Fixing the import resolved the error.

Q4: How does the ScrollToTop component improve user experience during navigation transitions?

In single-page React apps, when a user scrolls down on the home page and clicks a link to the About page, the browser keeps the vertical scroll offset from the previous page, leaving the user scrolled halfway down the new page. The ScrollToTop component listens to path modifications via the useLocation hook and executes window.scrollTo(0, 0), resetting the view back to the top on every transition.

Q5: How does the application support local offline or fallback testing if Supabase environment keys are missing?

The application checks if Supabase is configured via isSupabaseConfigured() in supabase.ts. If keys are missing, the client initializes as null. The services and the Auth Context intercept this state and fall back to local memory data providers and localStorage mock sessions. This allows developers to test the layout and transitions immediately without database configuration.
